# Skill: content-authoring

## 목적
문제 은행을 학습 의도에 맞게 일관되게 관리한다.

## 원칙
- 문제는 고정형으로 운영
- 난이도 하/중/상 기준 유지
- 공통분모는 12 이하 권장
- 상 난이도는 끊기 가능한 식 포함
- 모델 간 재사용 가능하게 메타데이터 정리

## 문제 추가 체크리스트
1. 식 원문 작성
2. 항 단위 정답 작성
3. 최종 정답 작성
4. 중간합 계산
5. 공통분모와 눈금 단위 계산
6. 권장 수직선 범위 계산
7. `QUESTION_BANK.md` 와 `problem-bank.json` 동시 수정

## 피해야 할 것
- 공통분모가 지나치게 큰 문제
- 이야기 모델에서 지나치게 어색한 맥락
- 토끼 모델에서 끊기 지점이 모호한 식
