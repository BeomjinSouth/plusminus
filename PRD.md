# PRD — 점프 계산 단일 모델 학습앱

작성일: 2026-03-23  
문서 상태: 단일 모델 운영 기준

---

## 1. 제품 정의

이 제품은 초등학생도 직관적으로 사용할 수 있도록
**점프 계산(토끼/부호-분해 모델) 하나만 남긴**
유리수의 덧셈·뺄셈 학습앱이다.

학생은 한 문제를 아래 순서로 푼다.

1. 식을 항 단위로 끊기
2. 각 항의 최종 부호 정리
3. 첫 번째 항 위치에서 시작해 나머지 항을 점프 이동하고 마지막에 최종 값 적기

---

## 2. 이번 방향 전환의 이유

- 셈돌 모델은 `-5-(+3)` 같은 유형에서 `0쌍 추가`가 필요해 직관성이 떨어진다.
- 점수 카드 모델은 연산 의미보다 화면 연출이 앞서 보일 수 있다.
- 현재 제품 목표는 **가장 설명이 쉬운 한 모델에 집중**하는 것이다.
- 따라서 MVP 는 `점프 계산`만 제공한다.

---

## 3. 목표 사용자

- 초등 고학년 ~ 중학교 초반 학생
- 괄호와 부호가 섞인 식을 처음 배우는 학생
- 긴 설명보다 짧은 단계와 직접 조작이 더 잘 맞는 학생

---

## 4. 핵심 학습 목표

- 식을 항 단위로 정확히 끊는다.
- 바깥 부호와 안쪽 부호를 합쳐 최종 부호를 정한다.
- 각 항의 절댓값만큼 수직선에서 이동한다.
- 마지막 위치를 최종 값으로 연결한다.

---

## 5. 화면 원칙

- 학생 화면에는 **문제 번호, 식, 해야 할 일, 필수 입력, 수직선**만 우선 노출한다.
- 통계 카드, 소개성 문구, 장황한 설명은 기본 화면에서 뺀다.
- 각 단계는 `지금 할 일` 한 장으로 시작한다.
- 단계 성공 시에는 짧은 축하 문구와 폭죽형 애니메이션으로 `맞췄다`는 사실을 즉시 보여 준다.
- 색만으로 의미를 전달하지 않는다.
- 모바일/태블릿에서도 같은 순서로 바로 풀 수 있어야 한다.
- **학생의 흥미를 유발할 수 있도록 생동감 있고 동적인 디자인(이모티콘, 부드러운 애니메이션, 밝은 색상)을 사용한다.**

---

## 6. 문제 운영 원칙

- 난이도는 `쉬움 / 보통 / 어려움` 3세트다.
- 각 세트는 3문항이다.
- 모든 문항은 고정형 문제 은행으로 운영한다.
- 필수 회귀 문항은 유지한다.
  - `-(+3)-(-5)+7`
  - `(+2/3)-(-5/6)-(+1/2)`
  - `(+2.4)-(-1.1)-(+0.5)`

---

## 7. 단계 구조

### 7-1. 끊기

- 학생은 항이 끝나는 곳만 눌러 식을 나눈다.
- 괄호 안은 끊지 않는다.
- 화면에서 첫 항의 바깥 괄호를 생략해 보여 준 경우에도 끊기 정답은 학생이 보는 식 기준으로 판정한다.
- 끊기 단계에서는 별도 `미리보기 식` 박스를 두지 않고, 원본 식 위의 끊기 위치 변화만으로 결과를 확인하게 한다.

### 7-2. 부호 정리

- 각 조각마다 `+ / -` 버튼으로 최종 부호를 먼저 고른다.
- 숫자 칸에는 부호 없이 크기만 입력한다.
- 숫자 칸 아래에는 현재 시스템이 읽은 항을 바로 보여 주고, 마지막 칸에서는 `Enter`로도 제출할 수 있게 한다.
- 정리 이후 화면에 보이는 최종 식도 `-3+8-4`처럼 괄호 없는 정리된 부호 형태로 보여 준다.
- 최종 식 입력은 항의 순서와 최종 부호가 맞으면 `-4-3`, `2.4+1.1-0.5`, `-(3)+5+7` 같은 동치 표기도 정답으로 인정하되, `-(-3)`처럼 괄호 안 부호 정리가 끝나지 않은 표기는 정답 처리하지 않는다.
- 최종 식 입력칸의 안내 문구나 placeholder 는 실제 정답을 미리 드러내지 않는다.
- 예: `-(+3)` → 부호 `-`, 숫자 `3`

### 7-3. 점프 이동 및 최종 값

- 음수는 왼쪽, 양수는 오른쪽
- 칸 수는 절댓값만큼
- 첫 번째 항 위치는 자동으로 출발 위치로 반영하고, 학생은 두 번째 항부터 순서대로 이동한다.
- 마지막 항의 이동이 끝나면 그 위치가 정답이 된다.
- 현재 항 정보는 `이번 항 / 방향 / 칸 수`로 단순 표시
- 마지막 도착 위치에서 최종 값을 입력하여 마무리한다.

---

## 8. 피드백 원칙

- 오답이어도 한 단계만 다시 시도하게 한다.
- 메시지는 짧고 직접적이어야 한다.
- 비난형 표현은 쓰지 않는다.
- 단계 성공 시 바로 다음 단계로 넘어간다.
- 마지막 정답이 맞았을 때는 다음 문제로 넘어가기 전에 축하 연출이 짧게 보이도록 한다.

---

## 9. 입장/플레이/결과 흐름

### 9-1. 입장

- 학교, 학년, 반, 번호 입력
- 세션 시작

### 9-2. 로비

- 학생 정보 한 줄
- `점프 계산` 제목
- `쉬움 / 보통 / 어려움` 시작 버튼

### 9-3. 플레이

- 상단: 문제 번호, 식, 해야 할 일, 진행 막대
- 본문: 현재 단계 카드
- 하단: 수직선 또는 입력 영역

### 9-4. 결과

- 완료 제목
- 정답 문항 수, 재도전 수, 최고 연속 정답
- 같은 세트 다시 하기
- 난이도 다시 고르기

---

## 10. 데이터/로그 원칙

- 모든 시도 기록은 append-only
- 단계별 재시도 횟수 보존
- 세트 완료 시 요약 결과 저장
- Google Sheets 는 서버에서만 호출

---

## 11. 비목표

- 셈돌 모델 제공
- 점수 카드 모델 제공
- 모델 잠금 해제 흐름
- 모델 간 추천 루프

---

## 12. 완료 정의

- 학생은 점프 계산 세트에 바로 진입할 수 있다.
- 셈돌/점수 카드 화면과 진입 경로가 제거되어 있다.
- 문제은행, 타입, 로그 스키마, 문서가 단일 모델 기준으로 정리되어 있다.
- 테스트와 빌드가 통과한다.

---

## 13. Fraction Rendering

- 분수가 포함된 식은 `2/3` 같은 평문 문자열 대신 분자/분모가 세로로 쌓인 실제 분수 모양으로 보여 준다.
- 이 규칙은 문제 헤더, 식 자르기 블록, 항 카드, 최종 식 리본, 수직선 눈금에 공통으로 적용한다.
